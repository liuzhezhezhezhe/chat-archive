# Contributing

[English](./CONTRIBUTING.md) | [简体中文](./CONTRIBUTING.zh-CN.md)

感谢你为 Chat Archive 做出贡献。

本项目的范围是刻意收窄的：它是一个本地优先、具备修订历史感知能力的 AI 聊天平台归档导出工具。最有价值的贡献，是强化这个核心方向，而不是把项目扩展成无关的工作流产品。

## 合规预期

贡献内容不得以帮助用户绕过平台政策、合同限制、同意要求、访问控制或适用法律为目的。

- 不要提交以规避平台反滥用系统或禁止性自动化规则为目的的改动。
- 不要优化隐藏式抓取、隐蔽账号轮换，或超出授权范围的数据访问能力。
- 优先接受能够提升透明度、用户控制能力、数据最小化和合法归档工作流的改动。

## 适合的贡献方向

- 修复 ChatGPT 或 Gemini 的 DOM 变化
- 提升抓取稳定性与可恢复性
- 提升导出 schema 的清晰度及配套文档
- 改进适配器架构
- 为新增 AI 聊天平台提供审慎、范围明确的支持
- 提升构建和打包脚本的测试或校验能力

## 通常需要强理由的改动

- 把项目改造成云服务或同步产品
- 增加账号系统或托管 API
- 与抓取/导出清晰性无关的大规模视觉重设计
- 让导出 schema 更难被程序消费的改动
- 主要目的是绕过平台限制或合规边界的功能

## 开发流程

1. 运行 `npm run doctor`。
2. 运行 `npm run build`。
3. 将 `dist/` 作为未打包扩展加载到浏览器中。
4. 在受支持平台上复现问题。
5. 用尽量小的改动修复根因。
6. 提交 Pull Request 前再次运行 `npm run build`。

## 实现说明

- 平台特定的 DOM 逻辑应尽量放在 `src/adapters/` 中。
- 调度和节奏控制逻辑应保留在 `src/background.js`。
- 页面交互流程应保留在 `src/content.js`。
- 除非有充分迁移方案，否则不要破坏修订历史感知导出模型。
- 不要在同一个改动里夹带无关重构。

## Pull Request 建议

- 说明用户可感知的问题是什么。
- 说明根因是什么。
- 说明为什么当前实现的范围是合适的。
- 只有当改动影响 popup 或 guide UI 时才附截图。
- 标明剩余风险，尤其是 DOM 选择器变更带来的风险。

## 平台支持预期

当前受支持的平台是：

- ChatGPT
- Gemini

如果你希望增加新平台，请优先沿用共享适配器模式，而不是把大量分支逻辑直接塞进 background 或 popup 层。
