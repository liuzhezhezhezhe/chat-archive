# Security Policy

[English](./SECURITY.md) | [简体中文](./SECURITY.zh-CN.md)

## 支持的安全问题范围

以下问题最符合本项目的安全报告范围：

- 导出归档中的非预期数据泄露
- 不安全的扩展权限或浏览器侧行为
- 可能导致已捕获会话内容泄露到本地浏览器环境之外的漏洞
- 受支持平台上的恶意页面交互路径

## 报告方式

如果你发现安全问题，不要在公开 issue 中披露可利用细节。

请通过你在 GitHub 上公布的私下安全联系方式联系维护者，并尽量附上：

- 问题简要说明
- 受影响版本或提交范围
- 复现步骤
- 影响评估
- 如果有的话，附上建议的缓解方案

## 合规边界

本项目仅面向已授权且合法的归档用途。

- 本软件无意授权任何违反平台条款、隐私义务、合同限制或适用法律的行为。
- 任何试图削弱用户同意、绕过访问控制或掩盖被禁止收集行为的请求，都应视为超出范围。
- 仅仅让抓取更激进的改动，并不自动构成安全改进，反而可能与项目的合规定位冲突。

## 当前设计约束

- 本项目是本地优先的，本仓库不包含服务端组件。
- 会话内容保存在浏览器扩展的本地存储中，并通过浏览器下载 API 导出。
- 扩展运行在受支持聊天平台的页面上，因此必须与页面 DOM 发生交互。

请将安全报告聚焦在真实的安全影响上，而不是泛化为抓取脆弱性。DOM 结构变化和选择器失效是重要 bug，但它们并不自动构成安全问题。
